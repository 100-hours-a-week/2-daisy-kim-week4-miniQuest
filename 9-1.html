문제 내용

let과 const의 재할당 차이점 파악하기
  1. let으로 선언된 변수 temperature에 처음에는 25를 할당하고, 이후에 30으로 값을 변경하세요.
  2. const로 선언된 변수 MAX_TEMPERATURE에 35를 할당하고, 이후에 이 변수에 다른 값을 할당하려 시도했을 때 발생하는 오류를 설명하세요.
  • [설명]
    const와 let의 차이는 변수에 다른 값을 다시 할당하려 할 때 확인할 수 있다.
    let은 선언한 변수에 값을 할당한 후 다시 다른 값을 할당할 때 문제 없이 가능하지만
    const는 재할당이 불가능해 TypeError가 발생한다.
  • 메모리 주소는 객체든 값이든 스택에 저장이 되는데 이를 변경하지 않게 설계함으로써
    코드에서 강제로 할당된 변수에 다른 걸 넣으려고 할 때 막아버린다.
    api 주소 같은 변경되면 안 되는 것들을 바뀌지 않게 함으로써 안전성을 보장하고
    메모리 할당, 해제를 다시 하게 하는 걸 막아 성능을 향상시킨다.
